# Creamy Singh — Handcrafted Ice Cream Cafe

Premium marketing site for Creamy Singh (Haridwar, Uttarakhand). Built with
**TanStack Start** (React 19 + Vite 7 + TanStack Router) and **Tailwind CSS v4**,
SSR-rendered, and deployable to Cloudflare Workers, Vercel, Netlify, or any
Node host.

## Requirements

- **Node.js 20+** (LTS recommended — see `.nvmrc`)
- **npm 10+** (bun and pnpm also work)

```bash
nvm use            # picks up .nvmrc -> Node 20
```

## Quick Start (Local)

```bash
npm install
npm run dev          # dev server (http://localhost:5173)
npm run build        # production build (.output/)
npm run preview      # serve the production build locally
npm run typecheck    # TypeScript check
npm run lint         # ESLint
npm run format       # Prettier
```

No environment variables are required. If you later add API keys, copy
`.env.example` to `.env` and fill in values. Variables prefixed with `VITE_`
are exposed to the client; everything else stays server-only.

## Project Structure

```
src/
  routes/                 # File-based routes (TanStack Router)
    __root.tsx            # Root layout: HTML shell, providers, error boundary
    index.tsx             # Home page (/)
  components/
    site/                 # Page sections (Hero, Menu, About, Visit, ...)
    ui/                   # shadcn/ui primitives
  assets/                 # Images, logo, menu pages
  lib/                    # Utilities, SSR error capture/page
  styles.css              # Tailwind v4 + design tokens (OKLCH)
  router.tsx              # Router factory
  server.ts               # SSR entry (Cloudflare Worker compatible wrapper)
  start.ts                # TanStack Start instance + middleware
public/                   # Static assets served at site root (favicon, etc.)
vite.config.ts            # Vite + TanStack Start + Cloudflare config
wrangler.jsonc            # Cloudflare Workers config
.nvmrc                    # Pinned Node major version
```

`src/routeTree.gen.ts` is auto-generated by the TanStack Router Vite plugin —
**do not edit it by hand**. Add or rename files in `src/routes/` instead.

### Why there is no `index.html` / `main.tsx` / `App.tsx`

This project uses **TanStack Start (SSR)**, not a classic Vite SPA. The HTML
shell, client bootstrap, and root component are all owned by the framework:

- HTML shell -> `shellComponent` in `src/routes/__root.tsx`
- Client/server entries -> generated automatically by the TanStack Start Vite
  plugin (virtual modules)
- Root component -> `RootComponent` in `src/routes/__root.tsx`

Adding `index.html` or `main.tsx` manually would conflict with the framework
and break SSR. This is intentional and correct.

## Deployment

`npm run build` emits both a static client bundle (`.output/public/`) and an
SSR server entry. The server entry (`src/server.ts`) is a thin wrapper around
`@tanstack/react-start/server-entry` that adds robust error capture and a
branded fallback page.

### Cloudflare Workers / Pages (default)
Configured via `wrangler.jsonc` (compatibility flag `nodejs_compat`).
```bash
npm run deploy        # vite build && npx wrangler deploy
```

### Vercel
Import the repo — Vercel auto-detects Vite.
- Build command: `npm run build`
- Output directory: `.output/public`
- Install command: `npm install`

### Netlify
- Build command: `npm run build`
- Publish directory: `.output/public`
- For SSR, use the Netlify adapter or deploy the Worker bundle as an Edge
  Function.

### Generic Node host
After `npm run build`, the `.output/` directory contains a Node-compatible
server bundle that can be run by any modern Node 20+ runtime.

## Tech Notes

- Design tokens live in `src/styles.css` — never hardcode colors in components.
- Section navigation uses smooth scrolling within a single landing route.
- No auth / database / external API calls — safe for any static-friendly host.
- SSR errors are captured by `src/lib/error-capture.ts` and rendered through
  `src/lib/error-page.ts` even when h3 swallows the original throw.
